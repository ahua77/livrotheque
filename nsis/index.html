Mais qu'est-ce ?

Livrotheque est un logiciel de gestion de sa bibliotheque personnelle. En gros ce logiciel permet d'enregistrer ses livres et de les classer suivant ses gouts.

Une petite liste de ce qu'il peut faire :

  • Tri des livres par titre/auteur/série ... tout ce qui vous passe par la tete
  • Insertion d'images de livres dans les fiches
  • Choix des colonnes affichées dans la liste des livres
  • Statistiques diverses sur la base de livre.
  • Remplissage d'une partie des renseignements à partir d'internet (dont l'image de la couverture)
  • Importation d'une base exterieure à partir d'un fichier CVS

26/01/2010 :
Une nouvelle version est disponible (1.2.0.0) faite par cubbisteIci

  • Cette version à donc été faite complètement par l'excellent cubbiste. La (longue!) liste de modifications
  • Ajout d'un bouton "fusionner deux groupes" qui ouvre une nouvelle boîte. Dans cette boîte, vous pouvez sélectionner deux groupes (genres, auteurs, éditeurs, ...) et les fusionner. Tous les titres du premier groupe sont rattachés au second, et le premier est retiré de la base.
  • Ajout d'un bouton "insérer par l'ISBN" (raccourci associé : F3 ou INS) pour saisir un livre, en étant directement placé sur le champ de saisie de l'ISBN
  • Ajout d'un message d'avertissement si vous faites une recherche par ISBN sur un code barre déjà présent dans votre base
  • Ajout d'un message d'avertissement quand vous validez la création/modification d'une fiche, si l'éditeur est inconnu dans la base.
  • Ce comportement est également disponible pour toutes les listes associées à un livre : auteur, format, pays, ...
  • Ce comportement est désactivable, liste par liste (oui, il est plus fréquent de créer un nouvel auteur qu'un nouvel éditeur, par exemple)
  • Ajout d'un raccourci clavier F2 pour saisir un livre
  • Possibilité d'utiliser la touche tab ou entrée pour se déplacer dans l'ordre logique de lecture dans la fiche de saisie d'un livre
  • Pour les statistiques, possibilité de n'afficher que les premières valeurs d'une série, en précisant le nombre de valeurs à conserver. Supposant que vous ayez 35 genres dans votre base, et que vous décidiez de n'en afficher que 20, les graphiques par genre présenteront 21 barres (ou 21 parts de camembert).
  • Possibilité de trier par date (de lecture, de publication, de mise à jour, ...) dans la liste principale : le tri se fait dans l'ordre chronologique.
  • Dans les listes (d'auteurs, d'éditeurs, de format, ...), possibilité de changer l'ordre des lignes en cliquant sur un entête de colonne pour choisir le critère de tri.
  • Ajout d'un bouton "paramétrer l'application" pour saisir les paramètres de l'application.
Pour plus de détails (entre autre sur les paramètres disponibles) allez sur le forum
